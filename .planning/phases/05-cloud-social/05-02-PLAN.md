  <name>Task 1: Create authService — Google Sign-In + Firebase Auth wrapper</name>
  <files>src/services/authService.ts</files>
  <action>
    Create `src/services/authService.ts` — a pure TypeScript module wrapping @react-native-google-signin/google-signin and @react-native-firebase/auth.

    **Design requirements:**
    - D-113 through D-118 (Google Sign-In with Web client ID, Firebase Auth credential exchange, silent sign-in, minimal OAuth scopes)
    - D-119 (authService separates SDK concerns from Zustand store)
    - All functions gracefully handle unconfigured Firebase (no crash if google-services.json missing)
    - Web client ID must be configurable (hardcode as a constant for now, flagged for env var in Phase 6)

    **Implementation:**

    ```typescript
    import { GoogleSignin, statusCodes } from '@react-native-google-signin/google-signin';
    import auth from '@react-native-firebase/auth';

    // D-114: Pass Web client ID (from Firebase Console → Web credentials)
    // IMPORTANT: Must be Web client ID, NOT Android client ID (D-114, P4)
    // Developer must replace this value with their Firebase Web client ID
    const WEB_CLIENT_ID = 'YOUR_WEB_CLIENT_ID_HERE';

    // D-117: Minimal OAuth scopes
    const OAUTH_SCOPES = ['profile', 'email'];

    /**
     * Configure Google Sign-In. Must be called once before any signIn/signOut calls.
     * Safe to call multiple times (GoogleSignin.configure is idempotent).
     */
    function configureGoogleSignIn(): void;

    /**
     * Attempt to sign in with Google.
     * Returns { user: { id, name, email, photo }, idToken } on success.
     * Throws typed errors:
     *   - SIGN_IN_CANCELLED: user cancelled
     *   - IN_PROGRESS: another sign-in is in progress
     *   - PLAY_SERVICES_NOT_AVAILABLE: Play Services not installed/updated
     *   - Any other error (wrapped): configuration issue (likely SHA-1 or Web client ID)
     * After GoogleSignIn success, exchanges idToken for Firebase credential.
     */
    async function signInWithGoogle(): Promise<{
      user: { id: string; name: string | null; email: string | null; photo: string | null };
      idToken: string;
    }>;

    /**
     * Sign out from both Google Sign-In and Firebase Auth.
     */
    async function signOutFromGoogle(): Promise<void>;

    /**
     * Attempt silent sign-in (restores previous session).
     * Call on app startup (D-118).
     * Returns user info if successful, null if no prior session.
     * Never throws — returns null on any failure (no prior sign-in, network, etc.).
     */
    async function signInSilently(): Promise<{
      user: { id: string; name: string | null; email: string | null; photo: string | null };
    } | null>;

    /**
     * Returns the currently signed-in Firebase user, or null.
     */
    function getCurrentUser(): { id: string; name: string | null; email: string | null; photo: string | null } | null;

    /**
     * Subscribe to Firebase Auth state changes.
     * Returns unsubscribe function.
     * Called when user signs in/out — fires on mount and on change.
     */
    function onAuthStateChanged(callback: (user: { id: string; name: string | null } | null) => void): () => void;
    ```

    **Key implementation details:**
    - `configureGoogleSignIn`: `GoogleSignin.configure({ webClientId: WEB_CLIENT_ID, offlineAccess: false, scopes: OAUTH_SCOPES })`
    - `signInWithGoogle`:
      1. Check Play Services: `await GoogleSignin.hasPlayServices({ showPlayServicesUpdateDialog: true })`
      2. `const { idToken, user } = await GoogleSignin.signIn()`
      3. `const credential = auth.GoogleAuthProvider.credential(idToken)`
      4. `const firebaseUserCredential = await auth().signInWithCredential(credential)`
      5. Return combined user data from Google + Firebase
    - `signOutFromGoogle`: `await GoogleSignin.signOut()` then `await auth().signOut()`
    - `signInSilently`: try `await GoogleSignin.signInSilently()`, exchange credential, return user info; catch all errors and return null
    - `onAuthStateChanged`: `auth().onAuthStateChanged(callback)`
    - Firebase Auth credential exchange happens AFTER GoogleSignIn success — this creates/links the Firebase account

    **Error handling:**
    - Catch `statusCodes.SIGN_IN_CANCELLED` — rethrow as user-friendly error
    - Catch `statusCodes.IN_PROGRESS` — rethrow
    - Catch `statusCodes.PLAY_SERVICES_NOT_AVAILABLE` — rethrow
    - All other errors: log and rethrow with "Google Sign-In failed — check SHA-1 fingerprints and Web client ID" message (D-113, P4 mitigation)

    **Note about Web client ID:**
    The string `'YOUR_WEB_CLIENT_ID_HERE'` is a placeholder. The developer MUST replace it with their actual Firebase Web client ID before the app will work. Document this clearly in comments.

  <name>Task 2: Extend authStore with Google Sign-In actions and wire authService</name>
  <files>src/stores/authStore.ts, src/services/index.ts</files>
  <action>
    Extend authStore (Zustand + AsyncStorage persist) with Google Sign-In actions that call authService.

    **Current authStore state:**
    ```typescript
    interface AuthStoreState extends AuthState {
      setPlayer: (playerId: string, playerName: string, token: string) => Promise<void>;
      signOut: () => Promise<void>;
    }
    ```

    **Extended interface:**
    ```typescript
    interface AuthStoreState extends AuthState {
      isAuthPending: boolean;          // true during sign-in/sign-out
      authError: string | null;        // non-null when last auth action failed
      setPlayer: (playerId: string, playerName: string, token: string) => Promise<void>;
      signOut: () => Promise<void>;
      googleSignIn: () => Promise<boolean>;    // new — returns success/failure
      googleSignOut: () => Promise<void>;      // new
      googleSignInSilently: () => Promise<boolean>;  // new — returns success/failure
      clearAuthError: () => void;               // new
    }
    ```

    **Implementation details (D-120 through D-123):**

    - `googleSignIn`:
      1. Call `authService.signInWithGoogle()`
      2. On success: extract `{ id: user.id, name: user.name, token: idToken }`
      3. Call `this.setPlayer(id, name, token)` for local persistence
      4. After persistence: import and call `syncQueue.drainQueue(processSyncEvent)` where `processSyncEvent` tries to submit each queued event via firestoreService (D-123)
      5. Set `isAuthPending: false`, `authError: null`
      6. On failure: set `authError: error.message`, `isAuthPending: false`, return false

    - `googleSignOut`:
      1. Call `authService.signOutFromGoogle()`
      2. Call `this.signOut()` for local persistence cleanup
      3. Set `isAuthPending: false`, `authError: null`

    - `googleSignInSilently`:
      1. Call `authService.signInSilently()`
      2. If returns user: call `this.setPlayer(user.id, user.name, 'silent_token')`
      3. After persistence: drain sync queue (same as googleSignIn)
      4. If returns null: silently return false (user never signed in before — expected)
      5. Never set authError on silent failure (not an error condition)

    - `clearAuthError`: set `authError: null`

    **Additional: Add auth service barrel export**
    - Update `src/services/index.ts` to export from `./authService` and `./firestoreService` and `./syncQueue`
    - Add authService, firestoreService, syncQueue exports alongside existing exports

    **Persist note:**
    AuthStore already persists to AsyncStorage via zustand persist middleware. The `isLoggedIn`, `playerId`, `playerName` fields persist across restarts. `isAuthPending` and `authError` should NOT be persisted (transient state) — configure persist to exclude them using `partialize`.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App ↔ Google Sign-In SDK | OAuth 2.0 — Google handles credential exchange |
| App ↔ Firebase Auth | Secure token exchange — Google idToken → Firebase custom token |
| Local ↔ AsyncStorage | Local auth token persistence — app-private storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-ID-05 | Spoofing | Google Sign-In | Mitigated | Google SDK handles token verification. Firebase Auth validates with Google's servers. Developer must register correct SHA-1 fingerprints (P4). |
| T-ID-06 | Tampering | Auth token in AsyncStorage | Accepted | AsyncStorage is app-private on Android. Token is Firebase OAuth token (not raw Google token). Rooted device attack surface accepted for MVP. |
| T-ID-07 | Information Disclosure | Player name in leaderboard | Accepted | Player chooses to sign in. Display name is Google profile name (can be changed). Email never displayed. |
| T-ID-08 | Repudiation | Sign-in cancellation | Accepted | User can cancel sign-in at any point. No gameplay impact. Analytics not implemented for MVP. |
</threat_model>
