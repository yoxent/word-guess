# Phase 6 — UI Design Contract

> Visual and interaction contract for the Pre-Launch & Polish phase. Generated by gsd-ui-researcher.

---

## 1. Phase Scope

This phase introduces the following UI elements and modifications to the existing React Native (Expo) word-guessing game:

| # | Element | Type | Requirement | Notes |
|---|---------|------|-------------|-------|
| 1 | **Color blindness texture overlays** | Additive to Tile | LAUNCH-01 | Dots (correct), diagonal stripes (present), solid fill (absent). Controlled by `colorBlindMode` toggle (OFF by default) |
| 2 | **Light / Dark / System theme** | Structural change | Polish | `colors.ts` → `lightColors` + `darkColors`. New `useColors()` hook. ~16 components migrate from direct import to hook |
| 3 | **Home screen stagger entrance** | New animation | Polish | Replaces 500ms setTimeout. Title → buttons (80ms stagger) → icon bar. RN Animated API. 300ms fade+slide |
| 4 | **How to Play modal** | New component | Polish | Modal overlay (not nav route). Tile examples + rules + "Got it!" dismiss. Triggered by `?` icon in Home top bar |
| 5 | **Sound service wiring** | Rewrite of existing | D-181–D-183 | `sound.ts`: no-op stub → expo-av `Audio.Sound`. Wire `playKeyPress`, `playReveal`, `playWin`, `playLoss` in components |
| 6 | **Reduce Motion toggle** | Conditional gating | LAUNCH-04 | Settings toggle (OFF by default). Skips all animations — tile flip, confetti, stat entrance, home stagger. User-controlled, not system-detected |
| 7 | **PixelRatio font scaling** | Constant-level change | LAUNCH-03 | `typography.ts`: all `fontSize` values × `PixelRatio.getFontScale()`. Tiles/layout NOT scaled |
| 8 | **TalkBack accessibility labels** | Additive props | LAUNCH-02 | `accessible`, `accessibilityLabel`, `accessibilityRole` on tiles, keyboard keys, buttons, modals |
| 9 | **BackHandler — centralized** | New pattern | LAUNCH-05 | Single `BackHandler.addEventListener` in Navigation.tsx. Blocked during animation/ad/IAP. Skip-to-final-state on back during animation |
| 10 | **Settings rows: 3 new** | Config extension | D-156, D-163, D-191 | `colorBlindMode` toggle, `reduceMotion` toggle, `themeSelector` (light/dark/system). Appended to settingsConfig in `config/ui.ts` |
| 11 | **Dead code cleanup** | Removal + fixes | D-178–D-180 | Delete ResultScreen.tsx. Fix white confetti (#ffffff → #f1c40f). Fix present tile/key text contrast (#ffffff → #1a1a2e) |
| 12 | **Performance markers** | Instrumentation | LAUNCH-07 | `console.time`/`console.timeEnd` at dictionary load, stats read, stats write. Guarded by `__DEV__` |
| 13 | **Privacy policy** | External page | LAUNCH-06 | Hosted on GitHub Pages (`docs/privacy.md`). Covers AdMob data collection, Google Sign-In data |

## 2. Design Tokens

### 2.1 Design System

| Property | Value |
|----------|-------|
| Tool | none (React Native StyleSheet.create) |
| Component library | none (custom RN components) |
| Icon library | `@expo/vector-icons` MaterialIcons (already established) |
| Font | System default (no custom font) |
| Animation engine | react-native-reanimated 4.x (tile flip, confetti) + RN Animated API (home stagger, stat entrance) |
| State management | Zustand 5.x with MMKV persistence |

### 2.2 Spacing Scale

No new spacing tokens. Phase 6 reuses the Phase 3 scale (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, keyboard key gap |
| md | 16px | Default element spacing (`screenPadding`: 6px for game, 16px elsewhere) |
| lg | 24px | Section padding, card edge padding |
| xl | 32px | Layout gaps between card groups |
| 2xl | 48px | Page-level top/bottom spacing |

**Exceptions:** Game screen uses `screenPadding: 6px` for more board space (inherited from Phase 2). Tile sizes are dynamic (cap 56px, floor 32px based on screen width + word length).

### 2.3 Typography

Extended from Phase 3's 5-size scale. Phase 6 adds `PixelRatio.getFontScale()` multiplier on all `fontSize` values, plus a new `Modal Title` (22px) token for the How to Play modal heading.

| Role | Base Size | After Scaling | Weight | Line Height | Usage |
|------|-----------|---------------|--------|-------------|-------|
| Stat Value | 32px | `Math.round(32 × scale)` | 700 | 1.1 | Big numbers on Stats screen |
| Card Title | 18px | `Math.round(18 × scale)` | 700 | 1.3 | Settings section titles, card headers |
| Settings Row | 16px | `Math.round(16 × scale)` | 400 | 1.5 | Toggle/row labels in Settings |
| Body | 14px | `Math.round(14 × scale)` | 400 | 1.5 | General body text, card descriptions, rules text in How to Play |
| Stat Label | 12px | `Math.round(12 × scale)` | 600 | 1.3 | Labels under stat values (uppercase recommended) |
| Modal Title | 22px | `Math.round(22 × scale)` | 700 | 1.2 | How to Play modal title only |

**Where scaling is NOT applied:**
- Tile letter font (computed dynamically: `tileSize × 0.48`)
- Tile/key dimensions and spacing
- Layout padding, gaps, margins

**Implementation:** `const fontScale = PixelRatio.getFontScale()` captured once at module load time in `typography.ts`. Each `fontSize` multiplied and `Math.round()`-ed. Acceptable for MVP — Android system font changes typically require app restart.

### 2.4 Color System

#### 2.4.1 Structural Change

`src/constants/colors.ts` is restructured from a single flat `colors` object into two named exports:

```typescript
export const lightColors = { ... };
export const darkColors = { ... };
```

## 3. Component Contracts

### 3.1 Tile.tsx — Additions

**Props (existing + new):**
```typescript
interface TileProps {
  letter: string;
  feedback: TileFeedback; // 'empty' | 'absent' | 'present' | 'correct'
  tileSize: number;
  index: number;
  isRevealing: boolean;
  isFirstRender: boolean;
  reduceMotion: boolean;            // NEW — skip animation when true
  colorBlindMode: boolean;          // NEW — render texture overlay
}
```

**States:**

| State | Visual | Accessibility Label | Texture |
|-------|--------|-------------------|---------|
| Empty (active row) | Gray border, no fill | `"Position {N}: empty"` | None |
| Filled (active row, not submitted) | Letter visible, no fill | `"Position {N}: {letter}, active"` | None |
| Revealing — Correct | Green background (#6aaa64/#66bb6a) | `"Position {N}: {letter}, correct"` | 3 white dots (triangle pattern) |
| Revealing — Present | Yellow background (#c9b458/#d4b84c) | `"Position {N}: {letter}, present"` | Diagonal white stripes |
| Revealing — Absent | Gray background (#787c7e/#636669) | `"Position {N}: {letter}, absent"` | Semi-transparent solid overlay |

**Interaction contract:**
- Tile is non-interactive (display only). `accessibilityRole="text"` not `"button"`.
- When `reduceMotion === true`: skip flip animation entirely. Show final state immediately.
- When `colorBlindMode === true && feedback !== 'empty'`: render absolute-positioned texture overlay View with `pointerEvents="none"`.
- Text color: `colors.keyText` for all states EXCEPT present, where it uses `#1a1a2e` (contrast fix D-180). In dark mode, present text same fix applies.

**Texture overlay implementation (absolute-positioned View, no SVG dependency):**
```typescript
// Correct: 3 small white circles in triangle pattern
<View style={[StyleSheet.absoluteFill, styles.textureContainer]} pointerEvents="none">
  <View style={styles.dotTop} />
  <View style={styles.dotBottomLeft} />
  <View style={styles.dotBottomRight} />
</View>

// Present: diagonal stripes (rotated container with narrow bars)
<View style={[StyleSheet.absoluteFill, styles.textureContainer, { transform: [{ rotate: '45deg' }] }]} pointerEvents="none">
  <View style={styles.stripeBar} />
  <View style={[styles.stripeBar, { top: '40%' }]} />
  <View style={[styles.stripeBar, { top: '70%' }]} />
</View>

// Absent: solid semi-transparent overlay
<View style={[StyleSheet.absoluteFill, { backgroundColor: 'rgba(0,0,0,0.15)' }]} pointerEvents="none" />
```

> **Important:** Apply `overflow: 'hidden'` on the parent Tile container View to prevent overlay clipping outside `borderRadius: 6`.

### 3.8 Navigation.tsx — Modifications

**BackHandler centralization (D-165–D-167):**

```typescript
// Wrapping the Navigation component or inside NavigationContainer:
useFocusEffect(
  useCallback(() => {
    const onBackPress = () => {
      const gameStore = useGameStore.getState();
      const adStore = useAdStore.getState();

      // During ad display: block back
      if (adStore.isAdShowing) return true;

      // During IAP: block back
      if (adStore.isIAPActive) return true;

      // During tile animation: skip to final state
      if (gameStore.isRevealing) {
        gameStore.skipToFinalState(); // completes reveals instantly
        return true; // consumed — let React Navigation handle after skip
      }

      // Otherwise: allow default back navigation
      return false;
    };

    BackHandler.addEventListener('hardwareBackPress', onBackPress);
    return () => BackHandler.removeEventListener('hardwareBackPress', onBackPress);
  }, [])
);
```

| Hard Mode toggle | 370ms | Same | 300ms | Same |

**Reduce motion:** All elements start at opacity 1, translateY 0. No animation runs.

### 5.2 How to Play Modal

- Open: `Modal` with `animationType="fade"` — backdrop fades in, card scales slightly (0.95→1.0) via RN Animated
- Close (tap backdrop or "Got it!"): `onClose()` — Modal handles fade-out automatically
- Sound: No sound on open/close

### 5.3 Texture Overlay Visibility

- Toggled via `colorBlindMode` in settingsStore
- When OFF (default): no overlay rendered, no performance cost
- When ON: overlay renders on every tile with non-empty feedback

### 5.4 Reduce Motion — Gating

| Feature | reduceMotion OFF | reduceMotion ON |
|---------|-----------------|-----------------|
| Tile flip animation | 200ms flip + 50ms stagger | Instant reveal (skip worklet animation) |
| Confetti | 40 particles, 1500ms | Not rendered |
| Home stagger | 300ms per element | Instant (all visible) |
| Stat card entrance | Fade-in animation | Instant |
| Sound | Played normally | Played normally (sound ≠ motion) |

### 5.5 BackHandler Skip-to-Final-State (D-167)

When user presses back during tile animation (`isRevealing === true`):
1. `gameStore.skipToFinalState()` is called
2. This sets all tile shared values to their final state (flipProgress=1, scale=1) with 0ms duration
3. `isRevealing` set to `false` immediately
4. Result state shown (win/loss modal)
5. Back action is consumed — user must press back again to navigate

This is safe because:
- Reanimated worklets can be interrupted by setting shared values to final values
- Skip animation is atomic (setTimeout/requestAnimationFrame to update store)
- No crash path — tiles always have a valid final state

### 5.6 Sound Playback

- All sound files loaded in `sound.init()` via `Audio.Sound.createAsync()`
- Playback uses `.replayAsync()` — reset + play in one call (faster than stop+play)
- Errors handled gracefully: `try/catch` with `console.warn` — no crash on missing sound file
- `soundEnabled` check in settingsStore gates the sound service's internal `_enabled` flag
- Files are static `require()` — Metro bundles them at build time

### 5.7 Theme Switching

- User selects theme via Settings segmented control (`light` / `dark` / `system`)
- Store persists `themeMode` to MMKV
- `useColors()` hook reactively returns the active palette
- All consuming components re-render with new colors on theme change
- React Navigation theme updates simultaneously (prevents white flash)
- `expo-status-bar` style switches simultaneously
- When in `system` mode: changing device dark mode preference while app is open triggers re-render via `useColorScheme()` reactivity

### 6.1 TalkBack — Tile Announcements (D-157–D-158)

| With letter | `"Position {N}: {letter}, {state}"` | `"Position 1: A, correct"` |

| Absent (gray) state label | `"absent"` | — |

**Implementation:** `accessibilityLabel` computed in Tile component from props, not stored in state.

### 6.2 TalkBack — Keyboard Key Announcements (D-159)

| Key | Label | Role |
|-----|-------|------|
| Letter A–Z | `"{letter}"` (uppercase) | `keyboardkey` |
| Enter | `"Enter"` | `keyboardkey` |
| Backspace | `"Backspace"` | `keyboardkey` |

### 6.3 Focus Management

- **Modal (HowToPlay / ContinueGame):** Focus trapped inside modal when open. `onRequestClose` handles Android back.
- **ResultModal:** Focus moves to modal when game completes. "Play Next" or "Back to Menu" receives focus.
- **Home screen stagger:** No focus management needed — elements appear sequentially but are all immediately focusable.
- **Settings toggles:** Switch components are natively focusable on Android.

### 6.4 Reduce Motion (D-163–D-164)

User-controlled via Settings toggle (OFF by default). Does NOT use `AccessibilityInfo.isReduceMotionEnabled()`. When ON:
- All `Animated.timing` calls bypassed (use `Animated.Value(1)` directly)
- Reanimated worklets: check `reduceMotion` prop, skip `withTiming`/`withDelay` if true
- Confetti component returns `null`
- Home stagger renders all elements at opacity 1, translateY 0

### 6.5 Color Blindness (LAUNCH-01)

Settings toggle (OFF by default). Texture patterns provide a second visual channel beyond color:
- Correct (green): 3 white dots in triangle pattern
- Present (yellow): diagonal white stripes
- Absent (gray): semi-transparent solid overlay

Not accessible via system accessibility setting — user must know to toggle in Settings.

### 6.6 Touch Target Sizes

All interactive elements meet minimum 44×44dp touch target (Android accessibility guideline):
- Tiles: ≥32px (floor) — at 32px, the touch target is small but acceptable (non-interactive tile does not require tap). Keyboard keys: dynamic flex layout ensures ≥44px height.
- Icon bar buttons: 40×40px with activeOpacity=0.7 — borderline but acceptable for top-bar icons (not primary interaction).
- Settings rows: full row height ~52px (8px padding + 16px label × 2).
- "Got it!" button: full modal card width, 14px padding vertical.

### 7.1 Empty States

### 7.2 Error States

| Scenario | Error Handling |
|----------|---------------|
| Sound file missing | `try/catch` in `sound.init()` → `console.warn`. Sound silently skipped. No crash. |
| Sound file corrupt | Same as missing — `Audio.Sound.createAsync()` throws, caught by try/catch |
| expo-av not available (web/expo go) | `try/catch` wraps entire `init()`. Sound disabled gracefully. |
| `useColorScheme()` returns null | Fallback to `'light'` in `useColors()` hook |
| Dark mode on unsupported Android version | `useColorScheme()` returns `null` → fallback to `'light'` (same as above) |
| BackHandler during animation + ad simultaneously | Check `isRevealing && !isAdShowing` → skip animation first |
| Theme switching mid-game | Tiles re-render with new colors via `useColors()`. No game state loss. |
| `colorBlindMode` toggled mid-game | Texture overlays appear/disappear on next render. No game interruption. |
| `reduceMotion` toggled mid-game | Existing animation in flight continues; next animation respects new setting |

### 7.3 Overflow / Truncation

| Element | Overflow Behavior |
|---------|------------------|
| How to Play "Got it!" button | `paddingHorizontal: 24` within max-width 85% card. Button text doesn't overflow. |
| Theme selector labels | "Light" / "Dark" / "System" — all ~6-7 chars, fit in segmented segments even at small widths. |
| Settings toggle descriptions | `"Shows patterns on tiles"` — single line, uses `body` (14px) with `flex: 1` on parent row. May wrap on very narrow screens (<320px). Acceptable. |
| Tile text (present fix) | Tile letter font size computed from `tileSize × 0.48` — single character, no overflow risk. |

### 7.4 Missing Data / Slow Network

| Scenario | Behavior |
|----------|----------|
| Sound files not yet provided (development) | `sound.init()` loads from `assets/sounds/`. If files don't exist, Metro will throw at bundle time. During development with missing files, developer must provide them. | 
| Privacy policy URL not reachable | Policy is linked from Play Store listing, not from within app. No in-app impact. |
| Performance markers in production | All `console.time`/`console.timeEnd` calls guarded by `__DEV__` — stripped from production AAB via Metro bundler dead-code elimination. |
