### B. Add performance markers to App.tsx
Add `console.time`/`console.timeEnd` around the dictionary loading code. In App.tsx, wrap the area after `isReady=true` where Navigation mounts. Since dictionary loading is synchronous via require() in dictionaryStore, add the marker at app initialization:

```typescript
useEffect(() => {
  if (__DEV__) {
    console.time('dictionary-load');
  }
  // All dictionary require() happens synchronously at module load, 
  // so the marker fires after module init completes
  fetchAdUnitIds();
  sound.init();
  sound.setEnabled(useSettingsStore.getState().soundEnabled);
  
  if (__DEV__) {
    console.timeEnd('dictionary-load');
  }
  
  setIsReady(true);
}, []);
```

### C. Add performance markers to StatsScreen.tsx
In `StatsScreen.tsx`, wrap the `statsStore.loadStats()` call (or wherever stats are loaded from SQLite). Find where stats loading happens and add:

```typescript
if (__DEV__) {
  console.time('stats-read');
}
// ... stats loading code ...
if (__DEV__) {
  console.timeEnd('stats-read');
}
```

### D. Add performance markers to GameScreen.tsx
In `GameScreen.tsx`, wrap the `statsStore.recordGame()` call (game completion stats write):

```typescript
if (__DEV__) {
  console.time('stats-write');
}
// ... recordGame call ...
if (__DEV__) {
  console.timeEnd('stats-write');
}
```

All markers are guarded by `__DEV__` — Metro's dead-code elimination strips these from production builds (LAUNCH-07).

Do NOT add any third-party performance monitoring tools. Visual FPS check is done manually (D-171).
