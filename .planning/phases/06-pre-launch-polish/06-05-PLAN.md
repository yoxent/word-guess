<action>
Migrate ALL screen files from the deprecated `{ colors }` import to `useColors()` hook. This is a mechanical change across 6 files.

**Standard migration pattern for each screen file:**

1. **Change the import:**
   - Before: `import { colors } from '../constants/colors';` (or relative variant)
   - After: `import { useColors } from '../hooks/useColors';`

2. **Add hook call** at the top of the component function (after other hooks, before return):
   ```typescript
   const colors = useColors();
   ```

3. **For style definitions:** Files that use `StyleSheet.create` with `colors.X` references need the StyleSheet moved inside the component, OR the specific dynamic colors need to be extracted as inline styles.

   The cleanest pattern is: keep `StyleSheet.create` for non-color styles, move color-dependent styles inline. Example:
   ```typescript
   // Before:
   <View style={styles.container} />
   
   // After:
   <View style={[styles.container, { backgroundColor: colors.background }]} />
   ```
   
   OR for simple cases, convert the entire StyleSheet to dynamic:
   ```typescript
   const styles = useMemo(() => StyleSheet.create({
     container: { backgroundColor: colors.background, ... },
   }), [colors]);
   ```

4. **For each file, verify** that every `colors.X` reference that was previously resolved at module level now resolves at runtime via the hook.

**Screen-specific notes:**

- **HomeScreen.tsx:** Already modified by 06-04 with stagger animation. Replace `import { colors }` with `import { useColors }` + `const colors = useColors()`.
- **GameScreen.tsx:** Simple — replace import + add hook call. Styles that use colors need inline conversion or useMemo.
- **StatsScreen.tsx:** Simple — same pattern.
- **SettingsScreen.tsx:** Simple — same pattern.
- **LeaderboardScreen.tsx:** Simple — same pattern.
- **LoadingScreen.tsx:** Simple — same pattern.

Do NOT modify app/Navigation.tsx or app/App.tsx — those are migrated separately in Task 1c (both modified by 06-04).
</action>
