<action>
For each file in the list, perform the colors import migration:

**Standard migration pattern for each file:**

1. **Change the import:**
   - Before: `import { colors } from '../../constants/colors';` (or relative variant)
   - After: `import { useColors } from '../../hooks/useColors';`
   
   Exception: If the file also needs `lightColors`/`darkColors` for comparison (e.g., Navigation.tsx which checks `colors === darkColors`), keep that specific import:
   ```typescript
   import { darkColors } from '../../constants/colors';
   import { useColors } from '../../hooks/useColors';
   ```

2. **Add hook call** at the top of the component function (after other hooks, before return):
   ```typescript
   const colors = useColors();
   ```

3. **For function components** that are already hooks-compatible, this is a simple addition.

4. **For style definitions:** Files that use `StyleSheet.create` with `colors.X` references need the StyleSheet moved inside the component, OR the specific dynamic colors need to be extracted as inline styles. 

   The cleanest pattern is: keep `StyleSheet.create` for non-color styles, move color-dependent styles inline. Example:
   ```typescript
   // Before:
   <View style={styles.container} />
   
   // After:
   <View style={[styles.container, { backgroundColor: colors.background }]} />
   ```
   
   OR for simple cases, convert the entire StyleSheet to dynamic:
   ```typescript
   const styles = useMemo(() => StyleSheet.create({
     container: { backgroundColor: colors.background, ... },
   }), [colors]);
   ```

5. **For each file, verify** that every `colors.X` reference that was previously resolved at module level now resolves at runtime via the hook.

**Specific per-file notes:**

- **Tile.tsx:** `FEEDBACK_COLORS` constant and `styles.letter` use `colors.textInverse` (for present fix, see D-180 in 06-03) — these become dynamic. Move `FEEDBACK_COLORS` inside component or use inline. The `tileStyle` local is already dynamic.
- **GameBoard.tsx:** Simple — `colors.X` in StyleSheet. Convert to useMemo or inline.
- **Keyboard.tsx:** `KEY_COLOR_MAP` and `styles` use colors. Move inside component or use inline.
- **ResultModal.tsx:** `styles` use colors. Use inline styles or useMemo.
- **Confetti.tsx:** Simple — only styles use colors. Convert.
- **LengthPickerModal.tsx:** Simple styles migration.
- **Button.tsx:** Simple styles migration.
- **StatCard.tsx:** Simple styles migration.
- **SettingsRow.tsx:** Has ThemeSelectorRow which already uses useSettingsStore directly. The component body and ToggleRow use colors for styles.
- **HomeScreen.tsx:** Already modified in Plan 06-04. Replace `const colors` import with hook.
- **GameScreen.tsx:** Replace import + add hook call.
- **StatsScreen.tsx:** Replace import + add hook call.
- **SettingsScreen.tsx:** Replace import + add hook call.
- **LeaderboardScreen.tsx:** Replace import + add hook call.
- **LoadingScreen.tsx:** Simple — just styles.
- **Navigation.tsx:** Already uses `useColors()` from Plan 06-04 — verify the import path and remove the old `colors` import.
- **App.tsx:** If still using `colors` for StatusBar, switch to dynamic from theme. App.tsx already has `activeTheme` from Plan 06-04, so it doesn't need `colors` import.

**IMPORTANT:** After migration completes, verify that `grep -r "from.*constants/colors" src/ --include="*.tsx" --include="*.ts"` returns ONLY:
- `constants/colors.ts` itself

ALL other files should use `useColors()` from hooks. The deprecated `colors` alias in colors.ts itself is kept for reference but should have zero consumers outside the two files above.

  sound.init();

    console.timeEnd('dictionary-load');
  }
