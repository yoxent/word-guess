    ```typescript
    interface AdStoreState {
      interstitialLoaded: boolean;
      interstitialLoading: boolean;
      rewardedLoaded: boolean;
      rewardedLoading: boolean;
      gamesSinceLastAd: number;

      preloadInterstitial: () => Promise<void>;

      reset: () => void;
    }
    ```

    Implementation details:
    - `preloadInterstitial`: Creates `InterstitialAd.createForAdRequest()` (or use hook pattern), subscribes to LOADED/CLOSED events, stores ad instance reference. Sets interstitialLoaded on LOADED. Does NOT re-load if already loaded (D-104).
    - `preloadRewarded`: Same pattern for `RewardedAd.createForAdRequest()`. Preload on app launch (D-104).
    - `showInterstitial`: Returns false if not loaded. Shows ad, returns true. On CLOSED event, preloads next ad. Returns false if frequency threshold not met (D-88 gating done by caller).
    - `showRewarded(onRewarded)`: Returns false if not loaded. Shows ad. On EARNED_REWARD event, calls onRewarded callback. Returns true if shown.
    - `gamesSinceLastAd`: Incremented by GameScreen on game completion via `incrementGamesSinceLastAd()`. Reset after interstitial shown via `resetGamesSinceLastAd()`. Session-local, not persisted (per CONTEXT.md frequency section).
    - `reset`: Clears all state. Called when store is no longer needed.

    Ad unit IDs fetched from `remoteConfig` service at preload time. In development (`__DEV__`), use `TestIds.INTERSTITIAL` and `TestIds.REWARDED` directly (D-108).
    Use simple Zustand state for ref-counting, not a class. Store is the singleton.
    Export `useAdStore` via the Zustand pattern (`export const useAdStore = create<AdStoreState>()(...)`).

    IMPORTANT: The ad store wraps react-native-google-mobile-ads behind a Zustand interface. Components import `useAdStore`, not the ad SDK directly. This follows the service layer pattern from Phase 1 (D-103).
    
    The ad instances (InterstitialAd, RewardedAd) are module-level variables created during preload, stored outside Zustand state (they're not serializable). The Zustand state tracks loaded/loading boolean flags.

    CRITICAL: Use event listeners pattern for RewardedAd (not hook) since the store isn't a React component. For the hook-based approach, components that need ad lifecycle can use hooks directly if needed, but the store provides the singleton coordination layer.

    Handle errors gracefully — if an ad fails to load, set loaded to false and loading to false so callers can proceed without crashing.

    Create `src/services/remoteConfig.ts` (D-106 through D-109):
    ```typescript
    import { TestIds } from 'react-native-google-mobile-ads';
    import remoteConfig from '@react-native-firebase/remote-config';

    // Default test IDs used when Remote Config fetch fails or in dev (D-108)
    const DEFAULT_INTERSTITIAL_ID = __DEV__ ? TestIds.INTERSTITIAL : '';
    const DEFAULT_REWARDED_ID = __DEV__ ? TestIds.REWARDED : '';

    let fetched = false;

    export async function fetchAdUnitIds(): Promise<void> {
      try {
        await remoteConfig().fetchAndActivate();
        fetched = true;
      } catch {
        // Fallback to test IDs on failure (D-108)
        fetched = false;
      }
    }

    export function getInterstitialAdId(): string {
      if (__DEV__) return TestIds.INTERSTITIAL;
      if (!fetched) return DEFAULT_INTERSTITIAL_ID;
      try {
        return remoteConfig().getValue('admob_interstitial_id').asString() || DEFAULT_INTERSTITIAL_ID;
      } catch {
        return DEFAULT_INTERSTITIAL_ID;
      }
    }

    export function getRewardedAdId(): string {
      if (__DEV__) return TestIds.REWARDED;
      if (!fetched) return DEFAULT_REWARDED_ID;
      try {
        return remoteConfig().getValue('admob_rewarded_id').asString() || DEFAULT_REWARDED_ID;
      } catch {
        return DEFAULT_REWARDED_ID;
      }
    }
    ```
    (Remote Config keys per D-109: `admob_interstitial_id`, `admob_rewarded_id`, `admob_interstitial_frequency_override` optional)

    In `src/app/App.tsx`:

    - The adStore's preload methods call `getInterstitialAdId()`/`getRewardedAdId()` which use cached/fetched values
