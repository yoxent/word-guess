| Property | Value |

## Component Contracts

### 1. UI Configuration Registry — `src/config/ui.ts`

**Contract:** A single plain-TypeScript file exporting two arrays.

```typescript
// Stats screen card definitions
interface StatCardConfig {
  id: string;                    // unique identifier, e.g. "overview"
  title: string;                 // card heading text
  type: 'overview' | 'byLength' | 'guessDistribution';
  order: number;                 // display order (lowest first)
}

// Settings screen section definitions
interface SettingsSectionConfig {
  id: string;                    // unique identifier, e.g. "gameplay"
  title: string;                 // section heading text
  rows: SettingsRowConfig[];
}

type SettingsRowConfig =
  | { type: 'toggle'; id: string; label: string; description?: string; storeKey: keyof AppSettings }
  | { type: 'placeholder'; id: string; label: string; description: string }
  | { type: 'info'; id: string; label: string; value: string };
```

**Rules:**
- No heavyweight abstraction. Plain arrays, typed union.
- Screen components iterate and switch on `type`.
- Future phases append to config arrays — Phase 4 adds ad/IAP toggles, Phase 5 swaps placeholder to signInButton.
- Path: `src/config/ui.ts`.
- Store binding via `storeKey` string — screen reads/writes `useSettingsStore()` by this key.

### 2. StatsScreen — `src/screens/StatsScreen.tsx`

**Contract:** Replace placeholder with config-driven card layout.

| Property | Contract |
|----------|----------|
| Layout | `ScrollView` with `contentContainerStyle` padding of `screenPadding` (16px). Cards rendered in `order` sequence from config array. |
| Loading state | Show spinner (ActivityIndicator) centered while `statsStore.loadStats()` resolves. Use `colors.accent` for spinner. |
| Empty state | Single centered card: "No games played yet." in body text. Subtitle: "Complete a game to see your stats!" in secondary text. |
| Content state | Sequence of stat cards, each with consistent card styling. |
| Share button | Fixed-position or bottom-of-list button: `<Button title="Share Results" variant="primary" />`. Visible only when stats exist and last game data available. |

**StatCard component contract** (new, in `src/components/ui/`):

| Prop | Type | Required | Default |
|------|------|----------|---------|
| `title` | `string` | Yes | — |
| `children` | `React.ReactNode` | Yes | — |
| `style` | `ViewStyle` | No | — |

**StatCard visual contract:**
- Background: `colors.surface` (#ffffff)
- Border radius: 12px
- Shadow: `{ shadowColor: '#000', shadowOffset: { width: 0, height: 2 }, shadowOpacity: 0.08, shadowRadius: 8, elevation: 3 }` (React Native)
- Padding: `lg` (24px) all sides
- Margin bottom: `md` (16px) between cards
- Title: Card Title typography (18px bold, `colors.textPrimary`)
- Title has 12px bottom margin before children

**Card types and their content:**

| Card Type | Content |
|-----------|---------|
| `overview` | Two-row grid of stat values: **Total Games** / **Wins** / **Win %** / **Current Streak** / **Best Streak** — each as labeled number |
| `byLength` | Table: rows for each word length (5-10) with columns: Length, Played, Won, Win %. Sorted by length ascending. |
| `guessDistribution` | Bar chart via `react-native-chart-kit`. X-axis: guess attempt number (1-N). Y-axis: count of wins. Bars mint green, zero-count bars muted gray. |

### 3. SettingsScreen — `src/screens/SettingsScreen.tsx`

**Contract:** Replace placeholder with config-driven section list.

| Property | Contract |
|----------|----------|
| Layout | `ScrollView` with `contentContainerStyle` padding of `screenPadding` (16px). Sections rendered in array order. |
| Loading state | None needed — settings are sync-loaded from MMKV via Zustand persist middleware. |

**Settings section card contract:**

**Row types:**

| Row Type | Renders | Behavior |

| `placeholder` | Label (left) + chevron/hint text "Coming soon" (right, secondary color) | Non-interactive. No press handler. |
| `info` | Label (left) + value string (right, secondary color) | Read-only display. No press handler. |

**Settings sections defined (in config order):**

**Chevron/hint for placeholder rows:** Text "Coming soon" in `colors.textSecondary` (12px), right-aligned. Not a real navigation arrow — just text.

### 4. Share Results — Emoji Grid Generation

**Contract:** Pure function + UI button.

**Function:** `generateShareText(gameResult: GameResult): string`

Input shape (sourced from last completed game in stats or from gameStore):

```typescript
interface GameResult {
  mode: GameMode;
  word: string;
  attempts: number;
  won: boolean;
  maxAttempts: number;
  guesses: GuessFeedback[][];  // each guess's per-tile feedback
  date: string;                // ISO date string
}
```

**Output format:**

```
Word Guess - Daily Challenge
2026-07-05

🟩🟩🟩🟩🟩
5/6

Play Word Guess!
```

Rules:
- Header: `"Word Guess - {Mode Name}"` (Mode Name: "Free Play", "Random", "Daily Challenge", "Endless")
- Date line: formatted as YYYY-MM-DD
- Blank line
- One emoji row per guess: 🟩 for `correct`, 🟨 for `present`, ⬛ for `absent`
- If won: last row shows all green, attempt counter on next line: `"{attempts}/{maxAttempts}"`
- If lost: show all guess rows, then `"X/{maxAttempts}"`
- Footer: `"Play Word Guess!"`

**UI button contract:**
- Located at bottom of StatsScreen (below last card, inside ScrollView or fixed position)
- Uses existing `<Button title="Share Results" variant="primary" />`
- On press: calls `generateShareText()`, then `Clipboard.setStringAsync(text)`, then shows a brief toast/alert "Results copied to clipboard!"
- Toast: simple `Alert.alert("Copied", "Results copied to clipboard!")` or a lightweight custom toast — Phase 3 can use Alert for simplicity, no custom toast infrastructure needed.

### 5. Streak Display

**Contract:** Stats overview card shows combined streaks per D-76.

| Display | Data Source | Behavior |
|---------|-------------|----------|
| "Current Streak" | Stats overview card | Shows the last-played mode's current streak when viewing main stats. Per-mode streaks accessible via per-length table or future detailed view. |
| "Best Streak" | Stats overview card | Max of all per-mode max streaks. |

Per-mode streak tracking added to `statsStore` and `storage.ts` — each mode (daily, endless, free/random) maintains its own streak. Free Play and Random share same streak counter (D-74).

Simple table with 4 columns:

```
Length    Played    Won    Win %
```

- Column headers: 12px semibold, `colors.textSecondary`
- Row values: 14px regular, `colors.textPrimary`

Bar chart rendered with `react-native-chart-kit`:

- Chart container height: 200px
- X-axis labels: "1", "2", "3", ... up to max attempts for the player's typical word length
- Y-axis labels: integer counts (0, 1, 2, ...)
- Bar color: `colors.tileCorrect` (#6aaa64) for bars with count > 0
- Zero-count bars: `colors.tileEmpty` (#d3d6da) with reduced opacity (0.5)
- Bar border radius: 3px top corners
- Chart padding: 12px from card edges
- Background: transparent (card surface shows through)
- No grid lines — clean minimalist look

## Accessibility

| Element | ARIA / accessibilityLabel | Behavior |
|---------|-------------------------|----------|
| Stat value | `accessibilityLabel: "Total games: 47"` | Screen reader announces "Total games: 47" |
| Stat label | Part of parent accessible view | Group label+value in an `accessible` View |
| Card title | `accessibilityRole: "header"` | Announce as heading |
| Toggle Switch | `accessibilityLabel: "Hard Mode, enabled"` | RN Switch has built-in accessibility; add label |
| Share button | `accessibilityLabel: "Share results"` | Standard button label |
| Bar chart | `accessibilityLabel: "Guess distribution chart. {N} games won in {M} attempts."` for each bar | Chart-kit may need custom accessible wrapper |

**Focus management:**
- No custom focus management needed — ScrollView handles naturally
- Settings rows: first row in each section gets initial focus when section appears (native behavior)

**Screen reader scan order:**
1. Screen title (header)
2. Cards in visual order (config array order)
3. Within each card: title → content (left-to-right, top-to-bottom)

## Edge Cases

| Scenario | Handling |
|----------|----------|
| **Empty stats (no games played)** | StatsScreen shows single centered card: "No games played yet." (18px bold) / "Complete a game to see your stats!" (14px secondary). No chart, no share button. |
| **Stats loading (async SQLite query)** | Centered `ActivityIndicator` with `colors.accent`. Do NOT show empty state until query resolves. |
| **SQLite query error / DB not initialized** | Show error card: "Could not load stats." (body text) / "Pull down to retry." (secondary). Add pull-to-refresh on ScrollView. |
| **Zero wins but games played** | Guess distribution chart shows all bars at 0. Chart renders with axis labels but flat line. "Win %" shows 0%. Current streak shows 0. |
| **Single game played (1 win)** | Overview card shows: Total 1, Wins 1, Win % 100%, Streak 1, Best 1. Chart has single bar at attempt count 1. |
| **Very long streak (>99)** | Stat value container handles 3+ digits. No truncation — 32px font naturally accommodates. |
| **All word lengths played (5-10)** | By-length table shows all 6 rows, scrollable within card if needed. |
| **Settings toggles conflict** | No conflicts — each toggle is independent. Hard Mode applies globally across game modes (no mode-specific override in Phase 3). |
| **Share with no games played** | Share button hidden when stats are null or `totalGames === 0`. |
| **Share with win vs loss** | Different emoji grid footer: `"5/6"` for win, `"X/6"` for loss. Grid shows all guesses in order. |
| **Extremely long guess history (>100 games)** | Stats aggregated from SQLite queries — no performance issue. `react-native-chart-kit` limited to maxAttempts bars (max 11, for 10-letter word). |
| **App backgrounded while on Stats/Settings** | No special handling — Zustand persist middleware auto-saves settings. Stats re-queries SQLite on mount via `loadStats()`. |
| **First launch (no settings saved)** | MMKV persist returns defaults from Zustand: hardModeEnabled=true, soundEnabled=true, hapticEnabled=true, isPro=false. Toggles render at these values immediately. |
| **Settings modified during gameplay** | Settings changes take effect immediately for next game. Hard Mode toggle is read at game start in gameStore — changing mid-game has no effect on current session (Phase 2 behavior). |
